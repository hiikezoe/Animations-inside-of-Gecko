documentの特定

        // nsIDocument* document;
        nsAutoCString spec;
        document->GetDocumentURI()->GetSpec(spec);
        printf("document: %s\n", spec.get());
Firefox の起動オプションには、-layoutdebug というオプションが付いていて、 これを指定することで先に説明したアニメーションがたくさんついた chrome ウインドウではなく、 シンプルな chrome ウインドウで作成された layout 機能のデバッグ目的に特化したブラウザが立ち上がります。 これを使えば、Firefox の chrome 上のアニメーションに悩まされることはなくなります。 ただ残念ながら、この layoutdebug はマルチプロセスに対応していません。 マルチプロセスでのバグを直したりする場合には、やはり Firefox を使うことになります。 そういうわけで、自分が見ているアニメーションが、Firefox の chrome 上にあるものなのか、 それとも HTML ドキュメント上にあるものなのかを区別する必要が出てきます。 nsIDocument を手に入れるには 1) nsPresContext から Document() 2) nsIPresShell から GetDocument() 3) nsRefreshDriver から PresContext()->Document() 4) RestyleManager から PresContext()->Document() 5) Animation から 　 GetRenderedDocument() (注) Gecko の命名規則で、getter に Get と付いている関数は nullptr を返す可能性があるという ドキュメントを読んだ記憶があるんですが、この資料を作成する時に探してみたら見つかりませんでした。 というわけで、適宜 nullptr チェックしてから使ってください。 ここに挙げた関数でいうと、nsRefreshDriver の PresContext() に nullptr チェックが必要になります。 あとは、Animation の GetRenderedDocument() にも必要です。 他は必要ありません。
